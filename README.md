# default_chatbot

Telegram-бот для автоматизации канала и связанного с ним чата обсуждений.

Бот подписывает опубликованные посты именем администратора, добавляет инлайн-клавиатуру со ссылками в комментариях и наводит порядок в чате (открепление сообщений, удаление стикеров, дедупликация комментариев для медиа-альбомов).

## Возможности

- **Автоподпись постов** - при публикации или редактировании поста в канале бот дописывает к тексту/подписи фиксированный "футер" (например, ссылку на канал).
- **Подпись автора в комментариях** - под форварднутым постом в чате обсуждений бот отвечает сообщением с именем администратора, который опубликовал пост.
- **Инлайн-клавиатура** - кнопки со ссылками (чат, буст, владелец и т.д.) добавляются к ответам бота.
- **Дедупликация альбомов** - если в посте несколько фото/видео (media group), бот отвечает в комментариях только один раз, а не по разу на каждый файл.
- **Автооткрепление и очистка** - форварднутые из канала сообщения в чате автоматически откручиваются от закрепа; стикеры, приходящие вместе с форвардом, удаляются.
- **Уведомления об ошибках** - при исключении в хендлерах бот присылает администратору сообщение с текстом ошибки.

## Стек

- Python 3
- [pyTelegramBotAPI](https://github.com/eternnoir/pyTelegramBotAPI) (`telebot`)
- `python-dotenv`

## Структура проекта

```
.
├── main.py            # точка входа: регистрация хендлеров и запуск polling
├── bot_instance.py     # создание объекта бота и обработчика ошибок
├── config.py           # загрузка и валидация переменных окружения
├── state.py             # общее состояние: дедупликация альбомов, подписи постов
├── handlers.py           # все обработчики сообщений и постов
├── keyboard.py            # инлайн-клавиатура с кнопками-ссылками
├── requirements.txt        # зависимости проекта
└── .github/workflows/       # автодеплой на сервер при пуше в main
```

## Установка и запуск

### 1. Клонировать репозиторий

```bash
git clone git@github.com:damnurmum/default_chatbot.git
cd default_chatbot
```

### 2. Создать виртуальное окружение и поставить зависимости

```bash
python3 -m venv venv
source venv/bin/activate   # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

### 3. Настроить переменные окружения

Создайте файл `.env` в корне проекта:

```env
BOT_TOKEN=токен_вашего_бота
CHAT=id_чата_обсуждений
CHANNEL=id_канала
ADMIN=id_администратора_для_уведомлений_об_ошибках
```

| Переменная | Описание |
|---|---|
| `BOT_TOKEN` | Токен бота, выданный [@BotFather](https://t.me/BotFather) |
| `CHAT` | ID чата обсуждений, привязанного к каналу |
| `CHANNEL` | ID канала, посты которого обрабатывает бот |
| `ADMIN` | ID пользователя, которому бот шлёт сообщения об ошибках |

> ID чата/канала можно узнать, например, через [@userinfobot](https://t.me/userinfobot) или пересылкой сообщения боту с `message.chat.id` в логах.

### 4. Запустить

```bash
python main.py
```

## Деплой на сервер

Проект настроен на автоматический деплой через **GitHub Actions**: при пуше в ветку `main` сервер (VDS) сам подтягивает изменения, обновляет зависимости и перезапускает бота через `systemd`.

Для работы автодеплоя на сервере должны быть настроены:
- `systemd`-сервис для бота (автозапуск и автоперезапуск при падении);
- скрипт `deploy.sh` (`git pull` → `pip install` → `systemctl restart`);
- секреты репозитория `VDS_HOST`, `VDS_USER`, `VDS_SSH_KEY` (Settings → Secrets and variables → Actions).
